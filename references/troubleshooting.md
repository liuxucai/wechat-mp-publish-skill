# 微信公众号发布 — 全流程问题与解决方法（实战记录）

> 周期：2026-07-16（导入）～ 2026-07-17（打通）
> 一句话结论：微信新版后台 DOM/坐标全变，技能里写死的旧坐标/选择器全部失效；叠加 React 受控组件不吃合成事件。一切问题最终靠「动态探测元素 + DOM trusted click + React 兼容 value setter」解决。

---

## 阶段一：技能导入与环境

### Q1. Windows 下 git 克隆被吊销检查拦截
- **现象**：从 GitHub `git clone` 直接失败。
- **根因**：Windows 凭据/吊销校验策略拦截。
- **解决**：用 Node.js 直接 `https` 下载仓库字节写入本地完成导入。

### Q2. lib.js 硬编码了错误的旧工作区 ID
- **现象**：脚本路径指向不存在的旧 workspace。
- **解决**：把 WORKSPACE / XB_PATH 等改成当前机器 `workspace-agent-d0d04e07`。

### Q3. 缺少 launch_chrome.cjs
- **解决**：参照 isolated-browser 启动器新建 `launch_chrome.cjs`（detached 拉起隔离 Chrome + 固定 profile + CDP 端口 9230 回退；优先复用 isolated-browser 的 9222）。

## 阶段二：封面选择与 AI 配图（核心难点）

### Q4. 封面未设时 AI 配图弹窗卡死 DOM
- **现象**：首篇发布时 AI 配图弹窗按钮语义从旧版「确认」变新版「使用/调整」，点「发表」被静默拦截，URL 停在 `action=edit`。
- **根因**：封面没设 → 微信拦截发表；弹窗按钮语义变了。
- **解决**：重走「选封面 → AI 配图 → 生成 → 使用 → 裁剪确认」完整链路。

### Q5. 全页只有一个 file input，属于正文「图片」按钮
- **现象**：`DOM.setFileInputFiles` 触发正文图片上传，封面拿不到入口。
- **根因**：封面上传做成菜单浮层，无独立 `<input type=file>`。
- **解决**：放弃文件上传路线，改走「AI 配图」生成路线。

### Q6. 封面菜单项坐标全返回 [0,0,0,0]
- **现象**：`getBoundingClientRect` 读菜单项坐标全是 0（隐藏渲染态）。
- **解决**：改用 `document.elementFromPoint` 纵向扫描定位真正可点的像素坐标。

### Q7. 点「使用」被 IMG 覆盖层拦截（z 序问题）❌→ 解法
- **现象**：CDP 坐标点「使用」落在封面预览 IMG 上，事件被吞。
- **解决**：改用 DOM `el.click()`（trusted 事件，不受覆盖层 z 序影响）。

### Q8. 封面「已设态 / 未设态」两套入口坐标不同
- **现象**：已设封面走「替换图标」、新草稿走「加号」，坐标不一致。
- **解决**：先探测封面状态（`.js_cover_preview_new` background-image），分别走替换图标 / 加号入口。**新版：统一用 `.js_cover_btn_area` 入口即可**。

### Q9. 旧封面入口类名 `.js_chooseCoverWrap`(918,453) 已不存在 ❌
- **现象**：点 (918,453) 是空气，菜单打不开。
- **根因**：新版微信重构封面区 DOM。
- **解决**：实测新入口是 `.js_cover_btn_area`，动态读中心坐标（约 629,313）。**该旧坐标已弃用**。

### Q10. AI 配图菜单项坐标写死错误 ❌
- **现象**：代码写 (827,558)/(964,628)，点不中。
- **解决**：扫描菜单里文字含「AI配图」的叶子节点动态取坐标。**旧坐标已弃用**。

### Q11. 对话框「是否打开」用 height 判定误判 ⚠️
- **现象**：`getBoundingClientRect().height` 常返回 0（transform/滚动），误以为弹窗没开。
- **解决**：可见性判定改为 `offsetParent!==null && display!=='none' && visibility!=='hidden'`（不看 height）。

### Q12. 「使用」按钮可见但滚出视口（y 为负）❌
- **现象**：新图在历史列表顶部，其「使用」按钮 y 坐标 -1842，CDP 点不到。
- **解决**：DOM `el.click()` 直接点该按钮（不依赖坐标/滚动）。

### Q13. 生成新图后「使用」误点历史旧图 ❌
- **现象**：封面用的是上一次历史图，不是本次新生成的。
- **根因**：「使用」按列表顺序取，没关联新图。
- **解决**：生成后拿到新图 src，向上 8 层 DOM 找**新图对应的** `.ai-image-op-btn` 精确 click。

### Q14. AI 配图对话框残留遮挡编辑封面弹窗 ⚠️（最关键）
- **现象**：点「使用」后 `.ai_image_dialog` 没关闭，盖住编辑封面的「确认」，确认点不中。
- **解决**：编辑封面「确认」改用 DOM click（trusted，不受遮挡）；确认逻辑不再依赖 `.cover-crop-con`，改为检测 `.weui-desktop-dialog` 是否 `display!=='none'`。

## 阶段三：裁剪框确认

### Q15. 裁剪「确认」坐标 (1013,825) 失效 ❌
- **现象**：点空气，裁剪框关不掉。
- **解决**：确认按钮是 `BUTTON.weui-desktop-btn_primary`，位于 `.weui-desktop-dialog__ft` 内。

### Q16. 确认点击后裁剪框仍 open ❌
- **现象**：坐标点击三次，弹窗判定 stillCrop 一直 true。
- **根因**：被残留 AI 配图对话框遮挡，CDP 点击落空。
- **解决**：DOM `el.click()` 点确认（见 Q14），重试 4 次 + 坐标兜底 (682,787)，确认弹窗真正关闭。

## 阶段四：发表链（原创声明 + 多级弹窗）

### Q17. 底部「发表」按钮坐标 (1312,920) 错 ❌
- **现象**：点空气，发表没触发。
- **解决**：实测 `button.mass_send`（约 1115,592）；`clickPublishBtn` 优先 DOM click `button.mass_send`。**旧坐标已弃用**。

### Q18. 正文 ≥300 字触发「原创声明」弹窗 ⚠️
- **现象**：点发表后弹「原创声明」，不处理就卡住。
- **解决**：`handlePublishConfirm` 开头加原创弹窗处理（填作者 + 勾选协议 + 点确定）。

### Q19. 作者字段 React 受控组件，value setter 无效 ❌→ 修正
- **现象**：`el.value=...` + dispatchEvent 填了「行」，React state 不更新，仍报为空。
- **解决**：用 `HTMLTextAreaElement.prototype` 的 value setter（React 兼容）；个别情况补 CDP 真实键盘输入。

### Q20. 原创弹窗关闭后需再点一次底部发表 ⚠️
- **现象**：关掉原创弹窗后没自动出发表确认。
- **解决**：原创处理末尾补一次 `cdpClick(1115,592)` 点底部发表。

### Q21. 发表确认弹窗嵌套两层 ❌
- **现象**：点「继续发表」后弹出真正的「发表」确认（带群发开关），旧脚本点到「继续发表」就 return。
- **解决**：`handlePublishConfirm` 改为循环：点「发表」(green primary) → 「继续发表」 → 「发表」，直到 URL 含 `appmsgid` 或微信验证。

### Q22. 「继续发表」坐标 (901,660) 不可靠 ❌
- **现象**：坐标点不中。
- **解决**：改扫 primary 类按钮 + DOM click 文字匹配，不依赖写死坐标。

## 阶段五：微信验证（人工环节）

### Q23. 微信验证弹窗阻断式，ESC/× 都关不掉 ⚠️
- **现象**：最后一步必须管理员扫码，弹窗无法被脚本关闭。
- **根因**：微信安全强制要求。
- **解决**：脚本识别到「微信验证」后提示用户扫码，轮询等待消失；消失后再点「继续发表」收尾。URL 出现 `appmsgid=...` 即发布成功。

## 阶段六：工程 hygiene

### Q24. 诊断脚本泛滥（累计 60+ 个）🧹
- **解决**：清理全部 `diag_*.cjs` 与历史截图，仅保留 `run_publish_yanfa.cjs`、`resume_publish.cjs` 及证据图。

### Q25. lib.js 单引号跨行字符串语法错误 ❌
- **现象**：修改后 `require` 报错 `Invalid or unexpected token`。
- **根因**：单引号 JS 字符串不能跨行。
- **解决**：内嵌脚本改用反引号模板字符串。

## 阶段七：Token 读取时机（本次发布前最新修复）

### Q26. getToken() 在导航微信首页前读取，误报「未登录」
- **现象**：用 `edit` 入口发「编辑」这篇文章时，脚本在 `getToken()` 阶段直接报错退出「未登录！」，但浏览器 profile 其实是已登录态（token=1744705079 有效）。
- **根因**：`getToken()` 解析的是**当前激活 tab 的 URL**。脚本启动时若当前 tab 是别的页面（如简书页、about:blank），`mp.weixin.qq.com` 的 token 就解析不到，于是误判未登录。这不是真的没登录，是读错 tab。
- **解决**：`publish.js` 主流程第 1 步先 `navigate('https://mp.weixin.qq.com/')`，等微信首页加载后**再**调 `getToken()`，确保从已登录的微信会话里取 token，然后才拼 `EDITOR_URL_BASE + '&token='` 跳编辑器。修复后同一次运行立刻跑通。
- **提醒**：若仍报未登录，确认隔离 Chrome 用的是带登录态的 profile（`~/.chrome_qclaw_stable`，CDP 9222），且微信首页确实载入了（不是卡在登录页）。

## 已弃用 / 行不通的方法（请勿再用）

| 方法 | 为什么不行 |
|------|-----------|
| 写死坐标 918,453 / 827,558 / 964,628 / 1013,825 / 1312,920 / 827,464 | 微信改版后 DOM 全变，全部失效 |
| `dispatchEvent(new MouseEvent(...))` 点击 | React 吃真实事件委托，合成事件被忽略 |
| `el.value = x` 给 React 受控组件赋值 | React state 不更新，需要用 prototype value setter |
| `getBoundingClientRect().height > 0` 判弹窗可见 | transform/滚动会让 height 返回 0，误判 |
| CDP 坐标点「编辑封面确认」 | 被残留 AI 对话框遮挡，点击落空 |
| `DOM.setFileInputFiles` 设封面 | 全页唯一 file input 属于正文图片按钮 |
| 按列表顺序取「使用」按钮 | 会误用历史旧图，须按新图 src 精确匹配 |
